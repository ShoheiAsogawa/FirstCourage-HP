# 株式会社ＦｉｒｓｔＣｏｕｒａｇｅ ホームページ戦略・設計・実装

## 1. サイト戦略の要約
本サイトは「催事買取でも安心して相談できる会社」として信用を形成し、第一CVである公式LINE登録・写真査定へ自然に誘導する集客サイトです。高額訴求やNo.1表現に頼らず、会社情報、古物商許可、査定だけでも相談可能、無理な営業をしない姿勢を要所で提示します。

## 2. ターゲット分析
第一ターゲットは40代〜60代女性。店舗に行く手間や買い叩きへの不安があるため、LINEで事前相談できる気軽さと、断ってもよい安心感が重要です。

第二ターゲットは50代〜70代男女。生前整理・遺品整理・断捨離で複数品目をまとめて相談したい層です。電話導線、会社概要、催事実績、許認可の見やすさが信用形成に効きます。

## 3. 競合参考から取り入れるべき要素の整理
参考サイトの強みは、上部・中部・下部の相談CTA、初めての方、買取方法、品目、ブランド、参考価格、FAQ、会社案内への導線が網羅されている点です。本サイトではそれを踏まえ、電話主導ではなくLINE査定を第一導線に再設計し、根拠管理が必要なNo.1表現や過度な高価買取訴求は避けます。参考元：[買取大吉](https://www.kaitori-daikichi.jp/)

## 4. サイトマップ
- `/` トップページ
- `/first-time` 初めての方へ
- `/purchase-methods` 買取方法
- `/items` 買取品目
- `/brands` 買取強化ブランド
- `/prices` 買取参考価格
- `/faq` よくある質問
- `/company` 会社概要
- `/contact` お問い合わせ / LINE案内
- `/columns` SEO記事一覧
- `/columns/[slug]` SEO記事詳細テンプレート

## 5. 各ページの役割一覧
トップページは信用形成とLINE登録の総合導線。初めての方へは不安解消。買取方法は催事・LINE・電話の選択支援。品目・ブランド・参考価格はSEOと相談前確認。FAQは離脱不安の解消。会社概要は怪しさの払拭。お問い合わせはCV専用。SEO記事は検索流入からLINE査定へ送客します。

## 6. 各ページの詳細構成
トップページ：FV、実績プレースホルダー、安心訴求、CTA、買取方法、品目、ブランド、流れ、参考価格、口コミ、催事実績、CTA、記事。

初めての方へ：不安の言語化、3つの約束、相談の流れ、準備物、CTA。

買取方法：催事買取、LINE査定、電話相談をセクション分けし、それぞれ安心ポイントを表示。

品目：カテゴリ一覧、写真の撮り方、CTA。

ブランド：強化ブランド一覧、ブランド不明品の相談導線、CTA。

参考価格：価格カード、表示方針、CTA。

FAQ：アコーディオン形式の質問回答、CTA。

会社概要：会社情報テーブル、許認可、実績プレースホルダー、CTA。

お問い合わせ：LINE査定のメリット、電話相談、CTA。

SEO記事：記事タイトル、キーワードタグ、本文、LINE CTA、関連記事。

## 7. 各セクションの見出し・本文・CTAコピー
主CTA：「LINEで写真を送るだけ査定」「LINEで無料査定」「公式LINEを追加する」

補助CTA：「電話で相談」「参考価格を見る」「LINE査定の流れを見る」

主要見出し：「使っていないブランド品を、まずは写真で気軽に相談。」「催事買取でも、相談先が見える安心を。」「初めての買取相談でも、急がず、わかりやすく。」

## 8. 信用を高めるための訴求設計
会社概要、古物商許可番号、所在地、代表者、電話番号をフッターと会社概要に明示。催事買取の不安には「運営元が確認できる」「査定だけでも相談可能」「納得できなければ売却不要」を繰り返し提示します。口コミ・実績は開発用プレースホルダーとして分離し、公開前に本番値へ差し替えます。

## 9. 公式LINE登録率を高める導線設計
FV、各ページ上部、買取方法、価格、FAQ、記事詳細、下部CTAにLINE導線を配置。LINEのメリットは「写真を送るだけ」「来場前に相談」「査定料無料」「しつこい営業なし」に統一します。

## 10. デザインコンセプト
白・明るいリネン背景をベースに、ネイビーで信頼感、落ち着いたゴールドで高級感を出します。ギラギラした演出や過度な赤・黄色の煽りを避け、読みやすい余白と大きめのCTAで高齢層にも配慮します。

## 12. フォント・UIトーン提案
本文は読みやすいゴシック体、見出しは明朝体で上品さを付与。角丸は8px程度に抑え、カードは情報整理用に限定。CTAは目立たせつつ、押し売り感を避ける文言にしています。

## 13. 必要な画像一覧
- ヒーロー画像：ブランド品とLINE査定の気軽さ
- LINE査定画像：スマホと品物写真
- 催事買取画像：明るい催事相談ブース
- 買取品目画像：バッグ・時計・ジュエリーのフラットレイ
- 会社信頼画像：書類・許認可・相談風景

## 14. 各画像の生成用プロンプト
生成プロンプトは `src/data/site.ts` の `imagePrompts` に一括管理しています。現在は生成済みのPNG画像を `public/images` に配置しています。

ブランド名買取、品目別査定、催事買取、LINE査定、初めての売却不安を主要クラスタにします。記事詳細からLINE査定へ送客し、品目・ブランド・参考価格ページへ内部リンクを伸ばします。

## 17. 実装技術の提案理由
Next.js App Router、TypeScript、Tailwind CSSを採用。Vercelデプロイとの相性が高く、SEOメタデータ、静的生成、共通コンポーネント化、データ分離を小規模サイトでも保守しやすく実現できます。

## 18. ディレクトリ構成
```txt
src/app              ルーティングとページ
src/components       ヘッダー、フッター、CTA、カードなど共通UI
src/data/site.ts     会社情報、ダミー実績、口コミ、記事、画像プロンプト
src/lib/seo.ts       メタデータとJSON-LD
docs                 戦略・実装ドキュメント
public/images        本番画像配置先
```

## 19. コンポーネント設計
`Header`、`Footer`、`CtaBand`、`PageHero`、`Section`、`InfoCard`、`ButtonLink`、`VisualFrame`を共通化。CTA、ページヒーロー、カードUIを再利用し、ページ追加時の品質差を抑えます。

## 20. 実装コード一式
実装コードはリポジトリ内に作成済みです。主要ファイルは `src/app/*/page.tsx`、`src/components/*.tsx`、`src/data/site.ts`、`src/lib/seo.ts`、`tailwind.config.ts` です。

## 21. ダミーデータ管理方法
実績数値、口コミ、電話番号、所在地、代表者、古物商許可番号、催事開催実績、参考価格は `src/data/site.ts` に集約しています。`placeholderNotice` と `note` で開発用プレースホルダーであることを明示しています。

## 22. GitHubに載せる前提の構成説明
Next.js標準構成のため、GitHubへそのままpushできます。`.env` が必要な外部APIは現時点でありません。LINE URLや本番ドメインは `src/data/site.ts` と `src/lib/seo.ts` で差し替えます。

## 23. Vercelデプロイ手順
1. GitHubにリポジトリを作成してpush
2. VercelでNew Projectを選択
3. Framework PresetはNext.js
4. Build Commandは `npm run build`
5. Output Directoryは未指定
6. 本番ドメイン設定後、`src/lib/seo.ts` の `siteUrl` と `metadataBase` を本番URLへ変更
